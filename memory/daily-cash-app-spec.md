# 데일리캐시 — 기획 설계서 (Claude Design용)

## 프로젝트 개요

**앱 이름 후보:** 데일리캐시 / 올캐시 / 머니루틴 / 돈모아
**한 줄 설명:** 골라서 돈 버는 앱 — 자고, 걷고, 쉬고, 찍으면 다 돈이 됨
**핵심 가치:** 유저가 자기 라이프스타일에 맞는 돈 버는 모듈을 선택, 하나의 앱에서 포인트 합산

### 포지셔닝
```
캐시워크       = 걷기만
돈이돼지       = 터치만
슬립머니       = 수면만
토스 만보기     = 금융앱의 부속 기능
────────────────────────────────
이 앱          = 전부 골라서 하나로 합산 → "돈 버는 원앱"
```

### 비즈니스 모델
```
광고주 (AdMob, 오퍼월 SDK)
        ↓ 광고비
앱 운영사 (50% 수취)
        ↓ 50% 포인트 배분
유저 (CU/스타벅스/메가커피 기프티콘 교환)
```

### 수익 시뮬레이션
| DAU | 일 평균 접점 | 일 광고노출 | 월 광고수익 | 유저 배분(50%) | 운영 수익 |
|-----|------------|-----------|-----------|--------------|----------|
| 1만 | 5회 | 5만 | ~750만원 | ~375만원 | ~375만원 |
| 5만 | 5회 | 25만 | ~3,750만원 | ~1,875만원 | ~1,875만원 |
| 50만 | 5회 | 250만 | ~3.75억원 | ~1.88억원 | ~1.88억원 |
| 300만 | 5회 | 1,500만 | ~22.5억원 | ~11.25억원 | ~11.25억원 |

※ eCPM $5 기준 (전면광고+보상형 혼합)
※ 핵심: 모듈 5개 활성화 시 접점 5회/일 → 캐시워크(1회) 대비 5배 광고 노출

### 경쟁 우위
| 기존 앱 | 하루 접점 | 일 수익(유저) | 이 앱 |
|--------|---------|-------------|------|
| 캐시워크 | 1회 | ~100원 | 5~8회, ~500원+ |
| 돈이돼지 | 수시 | ~200원 | 모듈 합산으로 상위호환 |
| 슬립머니 | 1회 | ~50원 | 수면 모듈 + 나머지 합산 |
| **합계** (3앱 설치) | 3회 | ~350원 | **1앱으로 500원+** |

## 타겟 유저

### 페르소나 1 — 앱테크 주부 (38세)
- 캐시워크 + 토스만보기 + 앱테크 3~4개 깔아둠
- 각각 쥐꼬리만큼 모이는데 관리 귀찮음
- "하나로 합쳐졌으면 좋겠다"

### 페르소나 2 — 대학생 (22세)
- 시험기간에 폰 안 쓰고 싶은데 의지력 부족
- "폰 안 쓰면 돈 준다고? 깔아야지"
- 디톡스 + 걷기 + 수면 모듈 조합

### 페르소나 3 — 반려동물 집사 (30세)
- 강아지 산책 매일 하는데 기록만 하고 보상 없음
- "산책하면 간식값 벌 수 있으면 좋겠다"
- 산책 + 수면 + 물마시기 모듈 조합

### 페르소나 4 — 직장인 (28세)
- 출퇴근 걷기 + 점심시간 디톡스 + 퇴근 후 영수증
- "자투리 시간에 용돈 벌기"

## 돈 버는 모듈 설계

### Module 1: 💤 수면 (Sleep-to-Earn)
```
방법: 취침 시 앱 활성화 → 가속도계+조도+마이크로 수면 검증
보상: 6시간 이상 수면 시 기본 포인트 (목표 달성 보너스)
접점: 기상 시 "수면 리포트 + 광고 + 포인트 적립" 화면
일 수익: ~80~120P
검증: 가속도계(움직임 감소) + 조도센서(어두움) + 마이크(코골이/정적)
```

### Module 2: 🚶 걷기 (Walk-to-Earn)
```
방법: 만보기 (헬스킷/구글핏 연동)
보상: 1,000보당 포인트, 일 상한 10,000보
접점: 목표 달성 시 알림 + 광고 + 포인트
일 수익: ~80~100P
검증: HealthKit/Google Fit API (OS 레벨 검증)
차별점: 캐시워크랑 같지만, 다른 모듈과 합산되니까 총 수익이 더 큼
```

### Module 3: 📵 디톡스 (Detox-to-Earn)
```
방법: "집중 모드" 시작 → 설정 시간 동안 폰 안 쓰면 포인트
보상: 30분 단위로 포인트 (최대 3시간/일)
접점: 디톡스 종료 시 "축하 + 광고 + 포인트" 화면
일 수익: ~60~150P (시간 비례)
검증: UsageStatsManager(Android) / Screen Time API(iOS)
킬러 훅: "폰 내려놓으면 돈 준다" — 역발상
```

### Module 4: 🐕 산책 (Pet Walk-to-Earn)
```
방법: "산책 시작" → GPS 추적 → 산책 완료
보상: 산책 거리/시간에 따라 포인트 + 주간 목표 보너스
접점: 산책 종료 시 "리포트 + 광고 + 포인트" 화면
일 수익: ~100~200P
검증: GPS 이동 궤적 + 속도(산책 속도 범위 확인)
확장: 산책 중 펫샵 쿠폰 푸시, 동물병원 예약 연계
```

### Module 5: 🧾 영수증 (Receipt-to-Earn)
```
방법: 영수증 사진 찍기 → AI OCR 자동 인식
보상: 영수증당 기본 포인트 + 특정 브랜드 상품 구매 시 보너스
접점: 촬영 후 "인식 결과 + 광고 + 포인트" 화면
일 수익: ~50~500P (구매 내용에 따라)
검증: OCR로 매장명/날짜/금액 확인 (중복 방지)
확장: 브랜드 제휴 시 아이템당 고액 캐시백 가능 (Ibotta 모델)
```

### Module 6: 💧 물마시기 (Water-to-Earn)
```
방법: 물 마실 때마다 "마셨어요" 탭
보상: 하루 8잔 목표 달성 시 포인트
접점: 탭할 때마다 짧은 광고 가능
일 수익: ~30~50P
검증: 솔직히 검증 불가 — 자기 신고 기반 (터치 인터벌로 최소 필터링)
참고: 검증 약하지만, 앱 접점 늘리는 역할 (하루 6~8회 추가 접점)
```

## 포인트 시스템

### 적립
```
1P = 1원 (직관적)
모든 모듈 포인트 → 하나의 지갑에 합산

일 최대 수익 예시 (모듈 6개 전부 활성화):
  수면:    100P
  걷기:    100P
  디톡스:  100P
  산책:    150P
  영수증:  100P
  물:       50P
  ─────────────
  합계:   ~600P = 약 600원/일 = 약 18,000원/월
```

### 교환
```
3,000P → CU 아메리카노 (2,000원) ← 실제 가치보다 낮게 교환 (마진 확보)
5,000P → 스타벅스 아메리카노
10,000P → 메가커피 1만원권
30,000P → 네이버페이 1만원
50,000P → 계좌 현금 인출 (1만원)
```

### 보너스 시스템
```
7일 연속 출석:     500P 보너스
30일 연속:        2,000P 보너스
친구 초대:        초대자/피초대자 각 1,000P
레벨업 보너스:    Lv.5/10/20/50 달성 시 특별 리워드
주간 챌린지:      "이번 주 5만보 달성" → 1,000P 추가
```

## 화면 구조 (Information Architecture)

```
[앱 열기]
    │
    ├── 홈 (수익 대시보드)
    │     ├── 오늘 수익 / 총 누적 수익
    │     ├── 활성 모듈 카드 (진행 상황)
    │     └── 오늘의 미션/챌린지
    │
    ├── 모듈 (돈 버는 활동)
    │     ├── 활성 모듈 관리
    │     ├── 모듈 스토어 (추가/제거)
    │     └── 각 모듈 상세 화면
    │
    ├── 지갑 (포인트)
    │     ├── 포인트 잔액
    │     ├── 적립 내역
    │     ├── 교환소 (기프티콘/현금)
    │     └── 출금 내역
    │
    ├── 랭킹 (소셜)
    │     ├── 전체 랭킹
    │     ├── 친구 랭킹
    │     └── 주간 챌린지 순위
    │
    └── 마이 (설정)
          ├── 프로필
          ├── 초대 코드
          ├── 알림 설정
          ├── 테마 설정 (컬러/다크모드)
          └── 고객센터
```

## Claude Design 프롬프트

### Wireframe 프롬프트:
```
모바일 앱 와이어프레임을 만들어줘.

앱 컨셉: "골라서 돈 버는 앱" — 사용자가 자기 라이프스타일에 맞는 돈 버는 모듈(수면, 걷기, 디톡스, 반려동물 산책, 영수증, 물마시기)을 선택하고, 모든 포인트가 하나로 합산되어 기프티콘/현금으로 교환하는 리워드 앱.

핵심 차별점: 기존 앱(캐시워크=걷기만, 슬립머니=수면만)은 하나만 되지만, 이 앱은 여러 모듈을 조합해서 총 수익을 극대화. "앱 5개 대신 1개로."

필요한 화면 7개:
1. 홈 — 수익 대시보드 (오늘 수익 + 활성 모듈 카드 2열 그리드 + 챌린지)
2. 모듈 스토어 — 활성 모듈 관리 + 추가 가능 모듈 리스트 (각 모듈에 예상 일수익 표시)
3. 수면 모듈 상세 — 수면 리포트 + 주간 기록 + 취침 모드 시작 버튼
4. 디톡스 모듈 상세 — 실시간 타이머 + 현재 적립 포인트 + 오늘 기록
5. 지갑 — 포인트 잔액 + 교환소(기프티콘 카드 그리드) + 최근 적립 내역
6. 온보딩 — 3스텝 (소개 → 모듈 선택 체크리스트 → 완료)
7. 설정 — 테마/알림/반려동물 정보/초대코드

하단 탭바: 홈 | 모듈 | 지갑 | 랭킹 | 마이

스타일: 미니멀, 깔끔, 와이어프레임(회색톤), 아이콘+텍스트, 각 모듈에 이모지 아이콘
```

### High-Fidelity 프롬프트:

위 와이어프레임을 기반으로 High-Fidelity 디자인을 만들어줘.

디자인 방향:
- 라이트 모드 기본 (배경 #F5F5F7, 다크모드도 지원)

## 기술 스택

### MVP
- **프레임워크:** React Native (Expo) 또는 Flutter
- **센서:** HealthKit(iOS) / Google Fit(Android) — 걷기, 수면
- **위치:** GPS — 산책 추적
- **카메라:** OCR — 영수증 인식
- **광고:** AdMob (배너+전면+보상형) + 오퍼월 SDK (TnkAd, NAS)
- **상태관리:** Zustand / Riverpod
- **로컬DB:** SQLite / AsyncStorage
- **인증:** Firebase Auth (소셜 로그인)

### 확장
- **서버:** Supabase (유저 데이터, 포인트 동기화, 랭킹)
- **OCR:** Google ML Kit / Naver Clova OCR
- **기프티콘:** 기프티스타 API / 쿠프 API (기프티콘 자동 발급)
- **푸시:** Firebase Cloud Messaging
- **분석:** Amplitude / Mixpanel
- **B2B 확장:** 브랜드 제휴 캐시백 (Ibotta 모델)
